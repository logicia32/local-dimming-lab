# local-dimming-lab

液晶（LCD）のバックライト **local dimming（ローカルディミング）** の効果と副作用を、
実機もデータシートも使わずに、たった 1 行の関係式から自分で描いてみる小さな実験道具です。

> **表示輝度 ＝ バックライト輝度 × 液晶の透過率**

液晶は自分では光らないので、後ろのバックライトを液晶のシャッターで遮って絵を作ります。
シャッターは完全には閉じきれない（漏れ光の下限がある）ため、黒が少し浮く。
local dimming はバックライトをゾーンに分けて暗いところだけ LED を絞り、黒を沈めて電力も下げます。
そのかわり **ブルーミング（ハロ）** という副作用がついてくる——その綱引きを、
モノクロ・1 フレームの最小モデルで観察します。

すべて合成シーン（数式で生成）なので、外部素材も著作権の心配もありません。
細部の数字は実パネルとは合いませんが、そのぶん種も仕掛けも全部見えます。

## できること

- バックライトのゾーン分割と、隣ゾーンへの光漏れ（ガウシアン拡散）をシミュレート
- **エッジライト（実質 1 次元）** と **直下型 / FALD（2 次元ゾーン）** の比較
- ゾーン数を振って、コントラスト・ブルーミング・省電力のトレードオフを観察
- 効果（深い黒・省電力）と副作用（ハロ・白飛び）を数値化（`metrics()`）

## 動かし方

Python だけで動きます（numpy / scipy / matplotlib）。

```bash
pip install -r requirements.txt
python scripts/make_figures.py
```

`outputs/` に 5 枚の図と `summary.txt`（各図の数値）が出ます。
`scripts/make_figures.py` の中で、ゾーン数・拡散の強さ（`spread`）・シーンを変えて遊べます。

最小の使い方:

```python
import numpy as np
from dimlab import simulate, reference, metrics, SCENES

target = SCENES["moon"]()                 # 黒地に明るい円（古典的なハロのデモ）
res = simulate(target, 16, 16)            # 16x16 = 256 ゾーンの直下型
print(metrics(res))                       # system_cr, leak_mean, power_saving, ...

edge = simulate(target, 1, 256, edge=True)  # 同じゾーン数のエッジライト（1 次元）
```

## このモデルが省いていること

仕組みを見るための最小モデルで、実機を再現するものではありません。

- **輝度だけ**を扱い、色は見ていません（モノクロ相当）。
- バックライトの広がりは**ガウシアンで近似**。実際の LED と導光・拡散はもっと複雑です。
- 「省電力率」は**バックライト指令値（LED の明るさ）の平均**から出した量で、実消費電力そのものではありません。
  LED 効率の非線形、ドライバの静的電力、制御のオーバーヘッドは無視しています。
- 時間方向（応答速度・ちらつき）も扱っていません。

それでも「黒がなぜ浮くのか」「local dimming は何を得て何を払うのか」
「エッジと直下で何が違うのか」「ゾーン数を増やすと何が起きるのか」——
このあたりの形と読み方は、ちゃんと見えてくると思います。
